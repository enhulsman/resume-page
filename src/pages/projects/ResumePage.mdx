# Portfolio Site

My personal site at [hulsman.dev](https://hulsman.dev). Dark-first design with warm amber accents, Syne + Outfit typography, scroll-triggered animations, and a hidden interactive terminal in the About section.

## What I Wanted

Full control over theming, layout, and deployment — not a template with someone else's opinions baked in. The site should feel like it was made by someone who works with terminals and infrastructure all day but also cares about how things look.

## Design

The palette is dark by default with warm amber/gold as the accent color. I specifically avoided the cold cyan-on-black and purple gradient aesthetics that scream "AI-generated." Syne as the display font gives headings a bold, geometric personality. Outfit handles body text — clean and readable without being boring.

The theme system runs on CSS custom properties. Switching between dark and light is instant — no layout shift, no flash. The light theme uses warm creams rather than stark whites.

## The Terminal

The About section has a terminal card that plays a typing animation on scroll. What most visitors won't notice: after the animation finishes, the blinking cursor is real. You can type into it.

It supports ~25 commands — `cat resume`, `ls projects/`, `neofetch`, `cowsay`, tab completion, command history with arrow keys, and a few easter eggs (`sudo hire-me`, `sl`, `matrix`). Navigation commands like `cd resume` actually route you to the page.

Built in vanilla TypeScript (~1200 lines), no framework. Mobile works through a transparent input overlay with touch gesture detection to distinguish taps from scrolls. Getting the keyboard to reliably open on Android and iOS took more iteration than I'd like to admit.

## Animations

Everything uses CSS keyframes or GSAP ScrollTrigger — no framer-motion or React animation libraries. The hero name has a character-by-character decode effect (scramble → settle) and a continuously shifting gradient. Sections fade in via GSAP as you scroll. The skills section has an infinite marquee ticker.

All animations respect `prefers-reduced-motion` and the site renders fully without JavaScript via Astro's static-first approach.

## Content System

Adding a project means creating one MDX file with frontmatter. Astro auto-discovers it, generates the route, and includes it in the gallery if `showcase: true`. Site-wide config (name, role, skills, social links) lives in a single `site.ts` file.

The contact form runs on Cloudflare Workers with the Resend API — server-side processing without a separate backend.

## Lessons Learned

**Don't hide content behind JS-dependent animations.** Early on I used Motion's `whileInView` which relies on IntersectionObserver. It silently failed in certain environments, leaving entire sections invisible. GSAP ScrollTrigger was far more reliable, and content is always visible by default now.

**Mobile input has sharp edges.** iOS won't open the keyboard for programmatic `.focus()` unless it's in a synchronous user gesture handler. Android doesn't reliably fire `input` events on elements with `opacity: 0`. A transparent overlay that captures focus also blocks scrolling. Each of these took a few rounds to get right.

**Monospace fonts lie about their width.** Hardcoding character width as `fontSize * 0.6` broke the terminal's `sl` train animation and `neofetch` layout. Measuring the actual rendered width of a character at runtime fixed it.
