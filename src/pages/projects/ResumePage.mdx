# Portfolio Site

A portfolio that treats design as seriously as infrastructure. Dark-first with a warm amber palette, editorial typography using Syne and Outfit, scroll-triggered animations via GSAP, and a fully interactive terminal hidden in the About section — complete with tab completion, command history, and easter eggs.

## Design Philosophy

The goal was a site that feels like it was made by an engineer who cares about craft. Not a template, not a generic "developer portfolio" — something with personality and intentional design choices.

**"Terminal Luxe"** — the intersection of a hacker's terminal and a luxury editorial. The dark background with warm amber/gold accents avoids the cold blue-on-black cliché. Syne as the display font brings a bold, chunky personality that immediately distinguishes it from the Inter/Roboto crowd. The terminal card in the About section takes this further — it's not just decorative, it's actually functional.

**Dark-first, but not dark-only.** The site defaults to the visitor's system preference, with a toggle for manual override. The light theme uses warm creams instead of stark whites, maintaining the same amber accent throughout.

## The Interactive Terminal

The standout feature: after the typing animation completes in the About section, the blinking cursor is actually interactive. Visitors who recognize it as a terminal prompt can type real commands.

**What you can do:**
- `help` — full command reference
- `cat resume`, `cat skills`, `cat education` — browse content without leaving the page
- `ls projects/` and `cat projects/<name>` — explore project details
- `cd resume` or `open contact` — actually navigate to other pages
- `neofetch` — ASCII art info card
- `sudo hire-me` — a fun easter egg with contact info
- `cowsay`, `fortune`, `sl`, `matrix` — developer humor

**Technical implementation:** The terminal is built entirely in vanilla TypeScript — no framework dependencies. It supports tab completion with longest-common-prefix matching, command history via arrow keys, and mobile keyboard input through a transparent overlay input element. The entire command registry, filesystem model, and rendering logic lives in a single file.

## Animation Approach

Animations serve the content, not the other way around. Every moving element has a purpose:

- **Text scramble on the hero name** — each character decodes from random symbols, reinforcing the technical identity
- **Shimmer gradient** — the name text continuously shifts through amber tones, adding life without distraction
- **Scroll reveals via GSAP ScrollTrigger** — sections fade in as you scroll, creating a sense of progression
- **Skills marquee** — an infinitely scrolling ticker that showcases the full tech stack at a glance
- **Orbiting dots** — subtle floating elements that make the page feel alive rather than static

All animations respect `prefers-reduced-motion` and degrade gracefully without JavaScript.

## Technical Highlights

- **Config-driven content:** `site.ts` defines personal info, skills, projects, and social links. MDX files are auto-discovered and routed — adding a project means creating one file
- **Edge deployment:** Cloudflare Workers handle the contact form server-side via the Resend API. No third-party form services, no exposed credentials
- **CSS custom properties:** The entire theme system runs on CSS variables, making theme switching instant with zero layout shift
- **Zero-JS static pages:** Astro's partial hydration means pages without interactive elements ship no client-side JavaScript at all

## Challenges & Lessons

**CSS animation reliability over JS animation.** Early iterations used framer-motion's `whileInView` for scroll animations, which silently failed when IntersectionObserver didn't trigger reliably. The lesson: for content that must be visible, never hide it behind a JS-dependent animation. The final implementation uses GSAP ScrollTrigger (battle-tested, framework-agnostic) and ensures all content is visible by default.

**Mobile input is harder than it looks.** Building the interactive terminal for mobile required solving several non-obvious problems: iOS refuses programmatic `.focus()` unless it happens in a user gesture's call stack, `opacity: 0` inputs don't reliably fire events on Android, and a transparent overlay that captures focus also blocks scrolling. The solution: `pointer-events: none` on the input with explicit `touchend` handlers that distinguish taps from scroll gestures.

**Typography needs breathing room.** Syne at extra-bold weights with tight tracking and `background-clip: text` clips the last glyph of each line. The fix was adding fractional em padding to the gradient text spans — a detail invisible to the eye but essential for rendering.
